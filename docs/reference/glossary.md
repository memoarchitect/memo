# Glossary

**Architecture-as-Code**
:   A way of representing architecture as structured, versioned, reviewable, and checkable source rather than only as static diagrams or documents.

**Architecture Element**
:   A traceable model element that represents part of the system architecture, such as a component, function, interface, behavior, hardware assembly, or physical element.

**Assurance Thread**
:   The connected path from need or risk to requirement, architecture, behavior, verification, evidence, and document view.

**Consistency Rule**
:   A model rule that checks for closure, coverage, lifecycle ordering, attributes, or cross-layer consistency.

**Document Projection**
:   A document or document section generated from, or backed by, model content.

**Evidence Element**
:   A model element representing evidence, such as a test result, report, analysis output, or controlled artifact.

**Hazard**
:   A potential source of harm. In meMO, modeled with typed attributes for hazard type, severity, and foreseeability.

**Methodology Profile**
:   A package of modeling practice, patterns, rules, gates, and viewpoints for a project or domain.

**Risk Control**
:   A measure intended to reduce risk, such as inherent safe design, protective measure, alarm, interlock, monitoring, or information for safety.

**Safety Thread**
:   A specific assurance thread focused on safety, usually connecting hazard, risk control, requirement, architecture, verification, and evidence.

**Semantic Link**
:   A typed relationship between model elements. meMO relationships such as `SatisfiedBy`, `VerifiedBy`, and `ProducesEvidence` specialize this concept. Every relation is a native SysML v2 `connection def`.

**Verification Case**
:   A verifiable element that defines how a claim will be checked, including method and acceptance criteria.

**View**
:   A concrete projection of model content for a specific purpose. Views make the model consumable without copying or forking it.

**Viewpoint**
:   A stakeholder-oriented perspective that defines what a view should address, per ISO/IEC/IEEE 42010.
